Rename Estimate Option
Last audited: May 6, 2026
Problem Statement
Renaming an estimate option today requires clicking a pencil icon directly on the document, which is buried and forces the user to leave the sidebar to do something that's fundamentally a navigation management action. Contractors building multi-option proposals need to label options meaningfully (Good/Better/Best, versioned quotes) without losing context in the sidebar.
States in this prototype
✅
Default — sidebar with option rows
Hover reveals triple-dot menu on each option row (desktop + mobile sheet)
✅
Dropdown open
Duplicate / Rename / Delete — Rename positioned between them
✅
Rename active — inline edit
Input replaces label, text selected, X and checkmark appear, drag handle hidden
✅
Confirm (Enter or checkmark)
Name saves, input collapses; syncs to document heading
✅
Cancel (Escape, X, or click outside)
Reverts to original name silently
✅
Empty field on confirm
Reverts to original name silently — no toast, no error
✅
Mobile — bottom sheet with keyboard
iOS keyboard slides up; sheet adjusts height; Done button on toolbar confirms
✅
Drag to reorder (desktop)
Option rows are draggable; a placeholder shows the drop target; row lifts with a shadow while dragged
States still needed
🔴
Save error
API PUT fails — sidebar shows new name but backend rejected it. Need visual recovery: auto-revert + inline error or toast.
High
🔴
Locked proposal — Rename absent
When status is LOCKED (sent/viewed), isEditingDisabled is true. Rename should not appear in the dropdown.
High
🔴
No edit permission — Rename absent
hasEditPermission / ProposalPolicy check gates all edit actions. View-only users should not see Rename.
High
🟡
Save in progress
isSaving state exists in OptionDisplay.tsx for other actions but not for rename. Need a subtle in-flight indicator on the checkmark/Done button.
Medium
🟡
maxLength: 80 vs 100 — needs alignment
QuoteDetailHeading.tsx uses maxLength=80. Backend NAME_MAX_LENGTH=100. This prototype uses 100. Engineering needs to align these before implementation.
Medium
🟡
Delete hidden when only one option
canDeleteOption = variations.length > 1 in OptionDisplay.tsx. Prototype shows Delete unconditionally.
Medium
⚪
Long name rendering at max length
PRD open question: does the document heading truncate or wrap at 80–100 chars? Needs a quick visual check at the extremes.
Low
⚪
Optimistic rollback on save failure
updateQuoteVariationFail() dispatch doesn't restore the sidebar label. Need design for the auto-revert path (also covers mobile offline scenario).
Low
