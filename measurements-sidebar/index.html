Measurements Sidebar
Last audited: May 6, 2026
Problem
Once a proposal is created, there's no way to add, swap, or edit the measurement data behind it. Contractors who start without a report, receive a corrected version, or need to override a specific value have no path — they either start a new proposal or live with the wrong numbers.

Decision log
🔵
Blank manual save = remove measurement
Saving the manual edit form with all fields empty or zero is valid — it acts as a detach, removing the measurement source from the proposal. No form validation error is needed. The confirmation modal copy should reflect this when all values are blank.
🔵
Load error falls back to the existing in-app pattern
No panel-specific error state is being designed. If the API fails on panel open, the standard full-page error treatment used across the app handles it. Engineering to confirm the fallback wires up correctly.
🔵
Unsaved changes dialog covers both panel close and page navigation
The "Discard unsaved measurements" modal fires for two triggers: closing the sidebar panel while in edit mode, and navigating away from the proposal page (e.g. clicking Back) while in edit mode. Same modal, same copy — the prototype was updated to remove the erroneous "?" from the title.
🔵
Zero-value measurements need no warning
A measurement field of 0 is valid and expected (e.g. a flat roof with no hips). No warning or indicator is needed in the sidebar or on mapped line items. Zero values flow through to quantities as-is.
🔵
Long report names truncate at dropdown width
No special handling needed. The source dropdown's full width is the natural truncation boundary — CSS text-overflow handles it. No additional design work required.
🔵
"In progress" reports are displayed but not selectable
A report that's still being processed (e.g. just ordered) appears in the State A picker with an orange "In progress" badge, a clock icon, and "Ordered [date]". It has no radio button and can't be selected. Design confirmed in Figma (node 53:43694).
🔵
Locked / read-only panel for sent proposals
When a proposal is sent or locked, the Measurements panel enters a fully view-only mode: the source dropdown is disabled (grey background, non-interactive), the Edit button is hidden, and measurements display as flat label/value rows. Design confirmed in Figma (node 549:13857).
States still needed
🟡
Load error
API fails when the panel opens. Defaults to the existing in-app full-page error treatment — no panel-specific design needed, but eng should confirm the fallback behavior.
🟡
Detach / remove measurement
Saving the manual form with all fields blank acts as a detach (removes the measurement source). There's no explicit detach control — if this is the intended path, it needs to be confirmed and the confirmation modal copy should reflect it.
⚪
Multiple structures
Jobs with more than one structure aren't handled. How does the panel show or let the user choose between per-structure measurement data?
